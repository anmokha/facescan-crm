Создание новой клиники

Добавление B2B клиента в систему

Пошаговая инструкция

1

Откройте Registry

Перейдите в /admin - увидите таблицу всех клиник

2

Нажмите "+ New Clinic"

Кнопка находится справа вверху страницы

3

Заполните форму

Clinic Name: Epilux Dubai
Slug (ID):   epilux-dubai (автоматически)
Owner Email: owner@epilux.ae
4

Создайте клинику

Нажмите "Create Clinic" - документ создастся в Firestore

Автоматические настройки

При создании клиники автоматически применяются следующие параметры:

{
  "name": "Epilux Dubai",
  "slug": "epilux-dubai",
  "email": "owner@epilux.ae",
  "plan": "trial",
  "theme": {
    "primaryColor": "#ec4899"
  },
  "limits": {
    "leads": 50
  },
  "status": "active"
}
✅
Готово к использованию
Клиника сразу доступна по ссылке: https://curescan.pro/checkup?client=epilux-dubai

Просмотр лидов

Мониторинг всех лидов системы

Global Leads View

Путь: /admin/leads

Доступные функции:

  • Фильтр по клинике (dropdown "All Clinics")
  • Сортировка по дате (новые → старые)
  • Просмотр статуса (new/contacted/booked/converted/lost)
  • Тип диагностики (skin/hair/nails)
  • Контактные данные (телефон, email)
ℹ️
Лимит отображения
Показываются последние 200 лидов (оптимизация производительности)

Структура данных лида

Поле Описание Пример
phone Телефон клиента +971501234567
email Email (опционально) client@example.com
status Статус обработки new, contacted, booked, converted, lost
diagnosticType Тип диагностики skin, hair, nails
clinicId ID клиники ABC123xyz
createdAt Дата создания 2025-01-02T10:30:00Z

Настройка брендинга

Персонализация внешнего вида

Изменение цветовой темы

1

Откройте Settings клиники

/admin/clinics/{id}/settings

2

Найдите секцию Theme

Поле "Primary Color"

3

Выберите цвет

#ec4899  → Pink (по умолчанию)
#3b82f6  → Blue
#10b981  → Green
#f59e0b  → Orange

Загрузка логотипа

В секции Theme есть поле logoUrl:

  • Загрузите логотип на хостинг (например, Cloudinary, S3)
  • Вставьте URL в поле
  • Логотип появится в Header чекап-страницы
⏱️
Кеширование
Изменения применяются через ~60 секунд из-за server-side кеширования

Настройка кастомного домена

Подключение собственного домена клиники

Пример: skin.epilux.ae

1

В админке CureScan

Зайдите в Settings клиники → Custom Domain

Custom Domain: skin.epilux.ae
☑️ Active
2

В DNS клиента

Попросите клиента создать CNAME запись:

Type:  CNAME
Name:  skin
Value: cname.vercel-dns.com
TTL:   3600
3

В Vercel Dashboard

Project Settings → Domains → Add Domain

Domain: skin.epilux.ae

Vercel автоматически выпустит SSL сертификат

4

Готово!

Клиент может открывать https://skin.epilux.ae

Middleware автоматически распознает домен и применит конфигурацию

⚠️
Время распространения DNS
Изменения DNS могут занять до 24-48 часов для полного распространения

Важные ограничения

Что нужно знать

Безопасность

⚠️
Super Admin доступ
Только admin@example.com и owner@example.com имеют доступ к GOD MODE.

Для добавления нового админа: app/admin/layout.tsx:9-12

Firestore Indexes

Некоторые сложные запросы требуют составных индексов:

  • Если видите ошибку "index required" в консоли
  • Откройте Firebase Console → Firestore → Indexes
  • Нажмите на ссылку из ошибки - индекс создастся автоматически
  • Ожидайте ~5-10 минут построения индекса

Кеширование

Client configs кешируются на сервере:

  • Время: 60 секунд
  • Зачем: Снижение нагрузки на Firestore
  • Следствие: Изменения видны не мгновенно (~1 минута)

Типичные задачи

Частые операции

Задача 1: Добавить нового клиента

  1. /admin → "+ New Clinic"
  2. Заполнить форму (Name, Slug, Email)
  3. Create Clinic
  4. Settings → настроить брендинг
  5. Отправить клиенту: /checkup?client={slug}

Задача 2: Клиент хочет свой домен

  1. Получить желаемый домен (например, scan.beauty.com)
  2. Settings → Custom Domain → указать домен
  3. Дать клиенту DNS инструкцию (CNAME → cname.vercel-dns.com)
  4. Vercel Dashboard → Add Domain
  5. Проверить через 24 часа

Задача 3: Увеличить лимит лидов

  1. /admin → найти клинику
  2. Settings → Subscription
  3. Plan: "pro", Leads Limit: 2000
  4. Сохранить

Задача 4: Проверить проблему клиента

  1. /admin → найти клинику
  2. Нажать Impersonate (синяя кнопка)
  3. Изучить dashboard от имени клиента
  4. Logout → войти обратно как admin
